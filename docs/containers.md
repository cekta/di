# Containers

**default:** []

В этом параметре перечисляются имена зависимостей которые должны быть доступны с помощью метода **get**, а также метод
**has** будет возвращать true только для контейнеров из данного списка.

В этом списке необходимо перечислить все зависимости которые вы планируете запрашивать с помощью **get**, обычно это
основные точки входа в ваше приложение(http обработчики, контроллеры, middleware, консольные команды и тд)

Библиотека на этапе компиляции пытается создать эти зависимости, а также разрешить все их дочерние элементы с помощью
других параметров конфигурации или механизма Autowiring в конструктор.

В случая наличие ошибок или неразрешимости какой-то из зависимостей вы получите соответствующую ошибку.

## Autowiring в конструктор

Autowiring в конструктор это автоматическое внедрение зависимостей на основе аргументов конструктора.  
Библиотека проходится по каждому аргументу, определяет имя зависимости которую надо внедрить в этот аргумент.

Если у аргумента указан тип и он не **buildin** такой как string, int и тд, то полное имя типа используется в качестве
имени зависимости, в остальных случаях используется имя аргумента без $

Каждая найденная зависимость анализируется дальше пока не разрешится вся цепочка зависимостей.  
Длинна цепочки (или вложенность зависимостей) никак не ограничена.

Механизм применяется ко всем containers по которым не задана специальная конфигурация с помощью [params](params.md)
или [alias](alias.md)

## [Пример autowiring в конструктор](https://github.com/cekta/di-example-usage/commit/94acea4bb96545c704c4b1b9c9a42ee2caa8b3fd)

```php
<?php

declare(strict_types=1);

namespace App;

class Example
{
    public function __construct(
        private A $a,
        private B $b,
    ) {
    }
}

class B
{
    public function __construct(private C $c)
    {
    }
}

class A
{
}

new \Cekta\DI\Compiler(
    containers: [\App\Example::class],
)->compile(); // Результат записать в нужный файл и можно пользоваться

```

Во время использования мы увидим что все необходимые аргументы корректно переданные, без дополнительной конфигурации.
